# Facebook Community Launch Plan

## Purpose

This playbook turns Facebook into the first broad-audience community channel for the Taifex Quant Trading Platform. The channel goal is education, trust building, demo traffic, and community discussion around Taiwan Index Futures quant infrastructure.

This plan does not automate Facebook account actions. Page creation, group creation, publishing, comments, and livestreams must be performed by an authorized human owner in Facebook after reviewing the copy.

## Channel Positioning

Facebook should be used for:

- public education on TX, MTX, and TMF workflows
- explanation of data governance, rollover, backtesting, and paper trading
- product demo clips for the Web Command Center
- community discussion with explicit safety boundaries
- routing interested users to the marketing website and controlled customer evaluation package

Facebook must not be used for:

- investment advice
- individualized futures recommendations
- live order instruction
- broker credential collection
- real account onboarding
- profit-promise or risk-minimizing language
- copy trading, managed account, signal subscription, or performance-fee promotion unless separately reviewed

### Facebook Group

Recommended name:

```text
台指期量化交易研究社
```

Recommended privacy:

```text
Private, visible
```

Rationale: visible discovery helps growth, while private posting allows moderation and reduces low-quality promotional comments.

Recommended description:

```text
本社團討論台指期 TX / MTX / TMF 的資料治理、回測、換月、紙上交易、風險控管與量化研究流程。社團不提供投資建議、不進行帳戶操作、不鼓勵實盤跟單。所有產品展示均為 Paper Only / research-only workflow。
```

Recommended member questions:

1. Do you understand that this group does not provide investment advice or live trading instructions?
2. Do you agree not to post profit promises, referral links, broker credentials, account screenshots, or personal financial data?
3. Which topic are you most interested in: data quality, rollover, backtesting, paper trading, risk controls, or Web Command Center demos?

## Group Rules

1. Educational discussion only.
2. No investment advice or individualized trade recommendations.
3. No profit-promise copy, screenshots used as claims, or signal-selling language.
4. No broker credentials, API keys, certificates, account IDs, or personal financial data.
5. No live order instructions or request for real account operation.
6. Product screenshots must be labeled Paper Only or research-only when order workflow appears.
7. Moderators may remove posts that blur the line between research infrastructure and regulated financial services.

## Human Review Workflow

Before publishing any post:

1. Check that the content is educational or infrastructure-focused.
2. Check that the post does not imply a trade recommendation.
3. Check that any product screenshot is labeled Paper Only when relevant.
4. Check that any strategy, backtest, or demo mention does not imply real performance.
5. Add the standard safety footer.
6. Confirm no customer data, account data, broker credential, or private key appears.

Standard safety footer:

```text
本內容僅供教育與產品展示，不構成投資建議。平台目前以 Paper Only / research-only workflow 為主，ENABLE_LIVE_TRADING=false，Live trading remains disabled by default.
```

## Moderation Escalation

Remove or hide comments that ask for:

- direct buy or sell instructions
- broker credentials, API keys, account IDs, or certificates
- real account operation
- copy trading
- performance guarantees
- managed account service

Reply template:

```text
我們目前僅討論研究、資料治理與 Paper Only demo workflow，不提供個別交易建議、跟單服務、代操或實盤委託操作。若你想了解平台功能，可以先看 Web Command Center demo 與客戶測試指南。
```

## KPI Dashboard

Track community growth and product interest, not trading outcome:

| KPI | Weekly target direction |
| --- | --- |
| Page followers | grow qualified audience |
| Group join requests | validate topic demand |
| Post saves | measure education value |
| Comment quality | identify research pain points |
| Website clicks | drive demo traffic |
| Demo guide opens | measure product interest |
| Customer evaluation requests | qualify leads |
| Unsafe comment removals | monitor compliance risk |

Do not use realized trading results, claimed profit, or user account performance as growth KPIs.
