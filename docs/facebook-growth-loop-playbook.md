# Facebook Growth Loop Playbook

All loops are content and education loops. They do not automate Facebook posting, create artificial engagement, collect credentials, provide investment advice, or enable live trading. Keep Paper Only, 不構成投資建議, and ENABLE_LIVE_TRADING=false visible.

## Loop 1: Content -> Web App Demo -> Evidence Sharing -> Community Discussion -> New Content

- Input: Education post about TX / MTX / TMF workflow or Paper Only demo.
- Action: Reader opens Web App, completes browser-only demo, copies evidence JSON.
- Conversion event: `demo_complete` or `evidence_copy`.
- Retention event: User posts a question in the Facebook Group.
- Metric: evidence_copy / Web App sessions.
- Weekly operating step: Turn common evidence questions into next week's demo posts.

## Loop 2: Free Checklist -> Waitlist -> User Interview -> Product Improvement -> Announcement

- Input: Checklist CTA from education post.
- Action: User asks for checklist or joins waitlist.
- Conversion event: `waitlist_signup`.
- Retention event: User completes interview and reviews updated demo copy.
- Metric: interview bookings per checklist request.
- Weekly operating step: Add interview insights to product copy and announce the improvement.

## Loop 3: Group Poll -> Product Demo -> Short Video -> Demo Traffic

- Input: Group poll on rollover, backtest, risk, OMS, or Paper Trading pain point.
- Action: Create a short demo clip showing the related Web App workflow.
- Conversion event: Web App link click.
- Retention event: User comments with a follow-up workflow question.
- Metric: demo_start from poll-derived content.
- Weekly operating step: Choose one poll answer to become a demo video topic.

## Loop 4: Professional User Feedback -> Templates -> New User Activation

- Input: Feedback from researchers, quant developers, or professional traders.
- Action: Convert feedback into templates, checklists, and example workflows.
- Conversion event: template download or evidence JSON copy.
- Retention event: user returns to test updated workflow.
- Metric: repeat Web App visits from community users.
- Weekly operating step: Publish one professional feedback-derived template per week.

## Loop 5: LinkedIn / B2B Thought Leadership -> Architecture Demo -> PoC

- Input: B2B architecture content about Trading OS, Risk Engine, OMS, audit, and hosted SaaS roadmap.
- Action: Route enterprise readers to official website and Web App architecture demo.
- Conversion event: demo request.
- Retention event: PoC scoping conversation.
- Metric: qualified B2B demo requests.
- Weekly operating step: Publish one architecture post and one follow-up case explanation.

Live trading remains disabled by default.
