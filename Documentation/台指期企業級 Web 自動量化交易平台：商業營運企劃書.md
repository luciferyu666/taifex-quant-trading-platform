# 台指期企業級 Web 自動量化交易平台：商業營運企劃書

## 1. 執行摘要與平台商業定位

在金融科技由「工具化」邁向「基礎設施化」的轉型浪潮中，本平台定位於從單一技術腳本進化為**量化交易作業系統（Trading OS）**。核心價值在於建構台指期交易的「底層架構」，而非僅是提供單一獲利策略。透過「訊號與執行解耦」的架構，平台實現了交易邏輯與經紀商通道的完全隔離，這不僅是技術上的優雅，更是商業上的風險對沖策略，確保企業客戶具備隨時切換券商接口的「業務連續性（Business Continuity）」。

以下為本平台作為企業級 Trading OS 與傳統 MVP 腳本的商業維度對比：

| 商業維度 | 傳統 MVP 腳本 | 企業級 Web 平台 (Trading OS) | 商業增益 (So What?) |
| --- | --- | --- | --- |
| **技術底座** | 本地 Python 環境 / SQLite | Kubernetes 容器化 / TimescaleDB / Kafka | 具備無限橫向擴展能力，支撐高併發多租戶 |
| **資料治理** | 零散 CSV 或單機 DB | Data Lakehouse (MinIO + ClickHouse) | 提供「機構級可重現性」，確保回測與實盤無落差 |
| **風險控管** | 腳本內寫死 Hard-coded | 多層級風控引擎 (Pre-trade / In-trade) | 建立合規護城河，防止極端市況引發之歸零風險 |
| **審計與稽核** | 文本日誌 (Local Log) | 不可篡改事件溯源 (Event Sourcing / WORM) | 滿足金融機構對於 RBAC 與稽核軌跡的剛性合規需求 |
| **擴張性** | 單人、單策略運作 | 多租戶架構、策略市集生態 | 從工具轉化為平台，降低邊際獲客成本 (CAC) |

## 2. 多元化營收模型與商業獲利路徑

為確保平台長期具備高經常性收入（ARR）並降低市場波動影響，我們設計了多層次、低合規風險的營收結構，優先專注於技術授權與資源計費。

### 2.1 核心 SaaS 訂閱模型

平台依據資料精度與運算配額，劃分四個主要訂閱等級。其中，Enterprise 版本定價 250k+ TWD，其商業邏輯建立在「高轉換成本」與「深度客製化 API 適配器」之上。

| 方案等級 | 目標客群 | 資料精度 | 核心功能 | 定價 (TWD/月) |
| --- | --- | --- | --- | --- |
| **Basic** | 量化初學者 | 分鐘級 (1m) | 基礎回測、模擬交易 (Paper Trading) | ~1,980 |
| **Pro** | 活躍交易者 | 秒級 (1s) | 實盤 API 接入、Webhook 風控告警 | ~8,800 |
| **Elite** | 專業交易員 | 逐筆 (Tick) | 多帳戶對帳、AI 策略健檢、雲端 Runner | ~29,800 |
| **Enterprise** | 機構/家族辦公室 | 全 L2 市場深度 | 私有雲部署、WORM 稽核日誌、SLA 保證 | 250,000+ |

### 2.2 資源使用量計費 (Usage-based Billing)

針對高階用戶的擴展需求，平台引進資源耗用計費，直接對應至平台運算成本，確保邊際貢獻：

- **AI 算力代幣 (AI Diagnosis Tokens)：** 用於「AI 策略診斷服務」，透過 LLM 偵測策略過度擬合 (Overfitting) 與參數敏感度分析，作為高利潤率之增值服務。
- **高頻資料重播 (Tick Replay)：** 針對 ClickHouse 引擎支撐的高頻逐筆資料回放計費，鎖定需進行 TCA (交易成本分析) 的專業用戶。

### 2.3 企業授權與導入服務

針對大型自營部，提供私有雲部署（On-premise / Private Cloud），透過 B2B 長期授權合約建立穩定的現金流支柱，並以此作為品牌背書。

## 3. 客群定位與精準定價策略

台指期市場（TX/MTX/TMF）具備獨特的階梯結構。本平台透過 **「風險等值正規化 (Risk Equivalent Normalization)」** 技術，將大台、小台、微台以 **1:4:20** 的曝險比例進行統一調度，這項專業功能是提升客群生命週期價值 (LTV) 的核心。

### 3.1 個人與專業交易員：定錨分析與轉化

- **定錨分析：** 參考國際競品（如 TradingView），本平台透過「在地化資料治理 (Local Data Optimization)」進行差異化定價。TradingView 缺乏台指期特有的「連續月價差調整 (Back-adjusted)」與「三大法人籌碼深度整合」，使本平台能支撐更高的溢價。
- **留存策略：** 針對 Basic 與 Pro 方案提供年繳折扣，透過交易日誌 (Trading Journal) 功能增加用戶數據積累，提升替換成本。

### 3.2 機構級用戶：治理為王

機構用戶對軟體的選擇權在於**風險治理能力**。Enterprise 版提供：

- **RBAC 權限矩陣：** 嚴格隔離研究員（策略開發）、交易員（執行監控）與風控官（Kill Switch 權限）。
- **高可用性 (High Availability)：** 基於 Kubernetes 的自我修復能力，確保交易通道 99.9% 以上的可用性。

## 4. 增值服務與生態系擴張策略

平台的終極目標是建立具備網路效應的生態系。透過策略市集與高品質資料服務，將平台從「單向工具」轉化為「雙向平台」。

### 4.1 策略市集 (Strategy Marketplace)

- **CAC 削減引擎：** 策略市集不僅是收入來源，更是**降低獲客成本 (CAC) 的關鍵**。透過第三方開發者自帶社群流量上架策略，平台可實現低成本用戶增長，並透過訊號訂閱分潤抽取 20%-30% 的平台費。

### 4.2 高價值資料 API

基於原始上下文，我們將資料治理成果獨立商品化。

- **連續月調整資料表：** 解決台指期換月跳空導致的回測偏差，提供「價差調整」與「比例調整」雙版本。
- **大額交易人籌碼 API：** 整合期交所之法人與大額交易人資料，銷售給需要多因子分析的研究機構。

## 5. 市場進入策略與長期競爭護城河

在全球量化市場競爭下，我們採取「專精台指期、深耕基礎設施」的差異化策略。

### 5.1 三階段成長路徑

1. **初期 (Tool-led Growth)：** 利用免費的高品質回測器與換月資料包吸納開發者。
2. **中期 (Ecosystem Expansion)：** 深度整合券商 API (如 Shioaji)，透過策略市集形成規模化。
3. **長期 (Infrastructure Moat)：** 建立包含私有雲部署與企業級風控標準的護城河。

### 5.2 長期護城河分析

1. **資料版本化 (Data Versioning)：** 基於 TimescaleDB 的時間序列治理，確保「今日回測之資料」與「實盤運行之資料」版本完全一致。這種 **機構級可重現性 (Institutional Reproducibility)** 是通用平台難以跨越的門檻。
2. **Broker Gateway 抽象層：** 支援多券商適配器，降低交易者對單一券商的依賴，建立中立平台信任度。

## 6. 合規邊界界定與風險治理商業框架

在金融科技領域，合規是**銷售加速器 (Sales Accelerator)**。我們透過嚴格的技術界定，將合規成本轉化為商業競爭力。

### 6.1 法規邊界與商業定位

- **SaaS 監管繞道：** 早期定位為純軟體技術提供者（SaaS），收取「技術服務費」而非「受託交易費」，以避開初期高額的牌照成本。
- **牌照轉型門檻：** 設定 AUM (管理資產規模) 或訂閱數臨界值，作為從技術平台轉型為受監管之期貨顧問/經理角色的戰略節點。

### 6.2 合規賦能與風險控管

- **安全標準加速銷售：** 系統對齊 **OWASP ASVS** 安全標準與期交所資料授權規範。這能大幅縮短企業客戶在採購過程中的「第三方風險評估」時間。
- **商業契約保障：** 在服務契約中明確定義「一鍵停機 (Kill Switch)」與「部位對帳 (Reconciliation)」的責任歸屬，透過技術透明度提升客戶的技術信任。

**結語：** 本商業企劃書不僅是一個技術方案的延伸，更是一個具備高度可規模化、合規敏捷性與技術護城河的金融科技版圖，旨在成為台指期量化市場的標準化作業環境。
